# Сценарии использования системы управления оборудованием

## 1. Управление инвентарем оборудования

### UC-1.1: Просмотр инвентаря оборудования
**Актор:** Администратор лаборатории  
**Предусловие:** Администратор авторизован в системе  
**Основной поток:**
1. Администратор переходит на вкладку "Инвенторизация"
2. Система отображает список всего оборудования с информацией о статусе, местоположении и владельце
3. Администратор просматривает информацию об оборудовании

**Альтернативный поток:**
- Администратор может использовать фильтры для отображения оборудования по статусу (доступно/выдано)

### UC-1.2: Добавление нового оборудования
**Актор:** Администратор лаборатории  
**Предусловие:** Администратор авторизован в системе  
**Основной поток:**
1. Администратор нажимает кнопку "Добавить оборудование"
2. Система открывает форму добавления оборудования
3. Администратор вводит данные об оборудовании (название, ID, группа, местоположение)
4. Администратор нажимает кнопку "Сохранить"
5. Система добавляет новое оборудование в инвентарь и отображает уведомление об успешном добавлении

**Альтернативный поток:**
- Если администратор нажимает "Отмена", система закрывает форму без сохранения

### UC-1.3: Фильтрация оборудования по местоположению
**Актор:** Администратор лаборатории  
**Предусловие:** Администратор авторизован в системе и находится на вкладке "Инвенторизация"  
**Основной поток:**
1. Администратор выбирает здание из выпадающего списка
2. Система обновляет список лабораторий в соответствии с выбранным зданием
3. Администратор выбирает лабораторию из обновленного списка
4. Система обновляет список комнат в соответствии с выбранной лабораторией
5. Администратор выбирает комнату из обновленного списка
6. Система отображает только оборудование, находящееся в выбранной комнате

**Альтернативный поток:**
- Администратор может сбросить фильтры, нажав кнопку "Сбросить"

### UC-1.4: Поиск оборудования по названию
**Актор:** Администратор лаборатории  
**Предусловие:** Администратор авторизован в системе и находится на вкладке "Инвенторизация"  
**Основной поток:**
1. Администратор вводит название оборудования в поле поиска
2. Система отображает оборудование, соответствующее введенному названию

**Альтернативный поток:**
- Если оборудование не найдено, система отображает сообщение "Оборудование не найдено"

## 2. Управление студентами

### UC-2.1: Просмотр списка студентов
**Актор:** Администратор лаборатории  
**Предусловие:** Администратор авторизован в системе  
**Основной поток:**
1. Администратор переходит на вкладку "Студенты и Доступы"
2. Система отображает список всех студентов с информацией о статусе доступа
3. Администратор просматривает информацию о студентах

**Альтернативный поток:**
- Администратор может использовать фильтры для отображения студентов по статусу доступа (есть доступ/нет доступа)

### UC-2.2: Добавление нового студента
**Актор:** Администратор лаборатории  
**Предусловие:** Администратор авторизован в системе и находится на вкладке "Студенты и Доступы"  
**Основной поток:**
1. Администратор нажимает кнопку "Добавить студента"
2. Система открывает форму добавления студента
3. Администратор вводит данные о студенте (имя, группа)
4. Администратор устанавливает статус доступа (есть доступ/нет доступа)
5. Администратор нажимает кнопку "Сохранить"
6. Система добавляет нового студента в список и отображает уведомление об успешном добавлении

**Альтернативный поток:**
- Если администратор нажимает "Отмена", система закрывает форму без сохранения

### UC-2.3: Изменение статуса доступа студента
**Актор:** Администратор лаборатории  
**Предусловие:** Администратор авторизован в системе и находится на вкладке "Студенты и Доступы"  
**Основной поток:**
1. Администратор нажимает на кнопку переключения статуса доступа рядом с именем студента
2. Система изменяет статус доступа студента
3. Система отображает уведомление об изменении статуса доступа

**Альтернативный поток:**
- Если происходит ошибка при изменении статуса, система отображает сообщение об ошибке

### UC-2.4: Поиск студента по имени
**Актор:** Администратор лаборатории  
**Предусловие:** Администратор авторизован в системе и находится на вкладке "Студенты и Доступы"  
**Основной поток:**
1. Администратор вводит имя студента в поле поиска
2. Система отображает студентов, соответствующих введенному имени

**Альтернативный поток:**
- Если студенты не найдены, система отображает сообщение "Студенты не найдены"

## 3. Отслеживание оборудования

### UC-3.1: Просмотр информации о выданном оборудовании
**Актор:** Администратор лаборатории  
**Предусловие:** Администратор авторизован в системе и находится на вкладке "Инвенторизация"  
**Основной поток:**
1. Администратор выбирает фильтр "Выдано" для отображения выданного оборудования
2. Система отображает список выданного оборудования с информацией о студенте, которому оно выдано, и дате выдачи
3. Администратор просматривает информацию о выданном оборудовании

**Альтернативный поток:**
- Администратор может использовать поиск для нахождения конкретного выданного оборудования

### UC-3.2: Возврат оборудования
**Актор:** Администратор лаборатории  
**Предусловие:** Администратор авторизован в системе и находится на вкладке "Инвенторизация"  
**Основной поток:**
1. Администратор находит выданное оборудование в списке
2. Администратор нажимает кнопку "Вернуть" рядом с оборудованием
3. Система обновляет статус оборудования на "доступно"
4. Система отображает уведомление об успешном возврате оборудования

**Альтернативный поток:**
- Если происходит ошибка при возврате оборудования, система отображает сообщение об ошибке

## 4. Сканирование QR/NFC

### UC-4.1: Сканирование QR-кода оборудования
**Актор:** Администратор лаборатории  
**Предусловие:** Администратор авторизован в системе  
**Основной поток:**
1. Администратор нажимает кнопку "Сканировать QR-код"
2. Система активирует камеру устройства
3. Администратор сканирует QR-код на оборудовании
4. Система распознает QR-код и отображает информацию об оборудовании

**Альтернативный поток:**
- Если QR-код не распознан, система отображает сообщение об ошибке

### UC-4.2: Просмотр информации после сканирования
**Актор:** Администратор лаборатории  
**Предусловие:** Администратор успешно отсканировал QR-код оборудования  
**Основной поток:**
1. Система отображает подробную информацию об оборудовании (название, ID, группа, местоположение, статус)
2. Администратор просматривает информацию об оборудовании

**Альтернативный поток:**
- Если оборудование выдано, система отображает информацию о студенте, которому оно выдано, и дате выдачи

## 5. Уведомления

### UC-5.1: Получение уведомления о выдаче оборудования
**Актор:** Администратор лаборатории  
**Предусловие:** Администратор авторизован в системе  
**Основной поток:**
1. Студент получает доступ к оборудованию
2. Система создает уведомление о выдаче оборудования
3. Система отображает уведомление в панели уведомлений
4. Администратор просматривает уведомление

**Альтернативный поток:**
- Администратор может отметить уведомление как прочитанное

### UC-5.2: Получение уведомления о возврате оборудования
**Актор:** Администратор лаборатории  
**Предусловие:** Администратор авторизован в системе  
**Основной поток:**
1. Студент возвращает оборудование
2. Система создает уведомление о возврате оборудования
3. Система отображает уведомление в панели уведомлений
4. Администратор просматривает уведомление

**Альтернативный поток:**
- Администратор может отметить уведомление как прочитанное

### UC-5.3: Получение уведомления об изменении контроля доступа
**Актор:** Администратор лаборатории  
**Предусловие:** Администратор авторизован в системе  
**Основной поток:**
1. Изменяется статус доступа студента
2. Система создает уведомление об изменении контроля доступа
3. Система отображает уведомление в панели уведомлений
4. Администратор просматривает уведомление

**Альтернативный поток:**
- Администратор может отметить уведомление как прочитанное

## 6. Управление местоположением

### UC-6.1: Организация оборудования по местоположению
**Актор:** Администратор лаборатории  
**Предусловие:** Администратор авторизован в системе  
**Основной поток:**
1. Администратор переходит на вкладку "Инвенторизация"
2. Администратор использует фильтры для выбора здания, лаборатории и комнаты

**Альтернативный поток:**

## 7. Статус оборудования

### UC-7.1: Просмотр статуса оборудования
**Актор:** Администратор лаборатории  
**Предусловие:** Администратор авторизован в системе и находится на вкладке "Инвенторизация"  
**Основной поток:**
1. Система отображает статус каждого оборудования (доступно/выдано)
2. Администратор просматривает статус оборудования

**Альтернативный поток:**
- Администратор может использовать фильтры для отображения оборудования по статусу

### UC-7.2: Просмотр деталей оборудования
**Актор:** Администратор лаборатории  
**Предусловие:** Администратор авторизован в системе и находится на вкладке "Инвенторизация"  
**Основной поток:**
1. Администратор находит оборудование в списке
2. Администратор нажимает на оборудование для просмотра деталей
3. Система отображает подробную информацию об оборудовании (название, ID, группа, владелец, местоположение, статус)
4. Администратор просматривает детали оборудования

**Альтернативный поток:**
- Если оборудование выдано, система отображает информацию о студенте, которому оно выдано, и дате выдачи
